# Contributing to VITA Frontier

VITA Frontier is infrastructure. Treat it like one.

## Ways to Contribute
- Bug reports with reproduction steps
- Tests
- Documentation
- UI/UX improvements that reduce friction
- Security reviews
- Adapter interfaces (not chain-specific implementations unless requested)

## Ground Rules
- Keep changes small and reviewable.
- Prefer clarity over cleverness.
- No proprietary lock-in proposals.
- No “tokenomics” proposals.
- No hostile feature-gating.

## Pull Requests
1. Open an issue first for non-trivial changes.
2. One PR should do one thing.
3. Include tests where possible.
4. Explain tradeoffs in plain language.

## Licensing
By contributing, you agree your contributions are licensed under AGPLv3.

## Ethos
Forking is allowed.
Capture is not.
Exit is sacred.
